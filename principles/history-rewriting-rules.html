History Rewriting Rules

Rewriting history (rebase, amend, force push) is powerful and dangerous. We allow it only in tightly defined situations.

Policy

  • Must not: Rewrite history on shared protected branches: main, release/*.
  • Must not: Force push to protected branches.
  • May: Rebase or amend commits on your own feature/* branch before review is complete.
  • Should: Prefer squash merge for integrating feature work, to keep main history clean without rewriting shared history.

Procedure

  1. If your feature branch is behind main, prefer rebase onto main before final review.
  2. Never rebase a branch that others are actively using unless explicitly coordinated.
  3. If you must force push your feature branch, announce it on the PR and ensure no one has based work on it.
