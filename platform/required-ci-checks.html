Required CI Checks

Minimum pipeline gates every repository must enforce before merging to protected branches, regardless of Git host.

Policy

  • Must: Block merges to main, release/*, and hotfix/* unless all required checks pass.
  • Must: Run required checks on pull/merge requests and default branch pushes.
  • Should: Fail the pipeline when coverage drops below the agreed project threshold.
  • Must not: Mark flaky or optional checks as required; instead fix or quarantine them.

Standard required checks

Quality & safety
  • Build: deterministic build, artifact signing where applicable.
  • Unit & integration tests: project-defined coverage threshold enforced.
  • Lint & formatting: language-standard linters (ESLint, flake8, golangci-lint, etc.).
  • Static analysis: SAST tools matching the stack (CodeQL, Semgrep, bandit).
Security & release gates
  • Dependency scanning: block on known critical/high CVEs.
  • Container/image scan: vulnerability scan for built images before publishing.
  • License compliance: fail on disallowed licenses.
  • Artifact provenance: sign images/binaries or produce SBOM for release artifacts.

Platform-specific mapping

GitHub
  • Mark workflow jobs as required in Branch protection â†’ Status checks.
  • Name the required checks identically to the workflow job (e.g., test, lint, container-scan).
  • Set Require branches to be up to date so checks rerun after rebase/merge.
  • Pin reusable workflows to a tag or SHA to prevent supply-chain drift.
GitLab
  • Set required jobs under Pipelines must succeed; ensure required jobs are not allow_failure.
  • Mark deploy jobs as protected and tie them to protected branches and environments.
  • Use rules: to block merge if required jobs are skipped or when [skip ci] is present.
  • Publish coverage values via coverage: regex and enforce thresholds in the job script.

Operational guidance

  • Document required check names in the repository README so contributors know what must pass.
  • Quarantine flaky jobs behind non-blocking labels while they are being fixed; do not mark them required.
  • For emergencies, merge via a hotfix/* branch with approval and a follow-up to restore failing checks.
  • Audit required checks quarterly to retire unused jobs and align with updated security baselines.
