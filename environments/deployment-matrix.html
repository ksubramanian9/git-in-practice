Deployment Matrix

A consistent view of what gets deployed where. The matrix below captures the expected signals, controls, and approvals per environment so teams can ship without guesswork.

Applies to: All services Goal: Predictable rollouts

Matrix

Environment Purpose Gates & checks Approval
Dev Developer integration and fast feedback. Unit tests, static analysis, container scan. Automated (CI passes).
QA / UAT Stabilization, exploratory testing, regression. E2E tests, contract tests, performance smoke. Release owner sign-off.
Prod Customer-facing or internal critical workloads. Change freeze window enforcement, monitoring hooks, error budget check. Change advisory or on-call approver.

Using the matrix

  1. Pin the deployment matrix in your service runbook; keep the table updated via PRs.
  2. Automate the gates with CI jobs and admission policies so approvals are recorded.
  3. Promote artifacts only when the prior environment meets all gates in the row above.
  4. For exceptions, log a ticket linked to the promotion so audit trails remain intact.

Signals to monitor

Quality
  • Test pass rates per stage
  • Coverage of critical paths
  • Performance regression delta
Reliability
  • Error budget burn rate
  • Change failure rate
  • MTTR for last 5 deploys
